# 登戸タイムズ 編集ガイドライン

記事を書くとき（人間・AIとも）は必ずこのルールに従うこと。

## 大原則: 事実のみを書く

- **Web検索・公式発表で確認できた事実だけを書く。憶測・創作・記憶頼みの記述は禁止。**
- すべての重要な事実（日付、店名、価格、施設規模など）は出典URLで裏付け、frontmatterの `sources` に記載する。
- 一次情報（公式サイト、プレスリリース、自治体発表）を優先する。まとめサイトのみが根拠の場合は「報道によると」「〜とのことです」と伝聞であることを明示する。
- 情報源同士で数字が食い違う場合は、より信頼できる方を採用し、断定を避ける表現にする。
- **日付の相対性に注意**: 「2026年春オープン予定」という古い記事を見つけても、現在すでにその時期を過ぎていれば「開業済みか」を必ず追加検索で確認する。予定の話と実績の話を混同しない。

## 情報源の監視

`data/watch-accounts.json` に監視対象の地域Instagramアカウントとローカルメディアのリストがある。毎回の記事探しで必ずチェックすること。

- Instagramは未ログインでは投稿を読めないため、次の方法で追う: (1) アカウント名や「登戸 イベント」「登戸 オープン」等でWeb検索 (2) 号外NET川崎市多摩区・タウンニュース多摩区版を確認（IG発のネタを数日以内に記事化しているため実質ミラー） (3) プロフィールページのWebFetchも一応試す
- 投稿の個別URL（instagram.com/p/xxx/）が判明した場合は記事に埋め込む

## ネタの優先順位

1. **お店の開店・閉店情報**（最優先。読者の関心が最も高い）
2. 施設・商業ビル・再開発の動き
3. イベント、行政サービス、交通の変化
4. その他の生活情報

開店情報はニュースサイトだけでなく、新店の公式Instagram・X投稿もWeb検索（例:「登戸 オープン instagram」「site:instagram.com 登戸 新店」）で探すこと。

## 画像・ビジュアルポリシー

記事にはできる限り視覚要素を1つ以上入れる。**使ってよいもの・ダメなものを厳守すること。**

### 使ってよい
- **公式X（Twitter）/ Instagram投稿の埋め込み**: 店舗や自治体の公式アカウントの投稿を、本文Markdown内にblockquote埋め込みHTMLで貼る（下記参照）。埋め込みスクリプトはサイト側が自動で読み込む
- **プレスリリース画像**（PR TIMES等、報道利用が認められているもの）: 画像をダウンロードして `public/images/posts/` に置き、frontmatterの `heroImage` に設定。`heroImageCredit` に「画像: ○○プレスリリースより」と出典を必ず明記
- **運営者が撮影した写真**: `public/images/posts/` に配置
- 自治体・官公庁が報道発表で提供している画像（出典明記）

### 使ってはダメ（著作権侵害・規約違反）
- 食べログ・Googleマップ・Rettyなどの口コミサイトの写真
- 他人のブログ・ニュースサイトの写真の転載（埋め込み機能がないサイトはNG）
- SNS投稿画像の直接ダウンロード転載（埋め込みは可、画像の切り出しは不可）

### 埋め込みの書き方（本文Markdown内にそのまま書く）

X投稿:
```html
<blockquote class="twitter-tweet"><a href="https://x.com/アカウント/status/投稿ID"></a></blockquote>
```

Instagram投稿:
```html
<blockquote class="instagram-media" data-instgrm-permalink="https://www.instagram.com/p/投稿ID/" data-instgrm-version="14"></blockquote>
```

### heroImage（アイキャッチ）frontmatter

```yaml
heroImage: /images/posts/2026-07-11-example.jpg   # または https:// のURL
heroImageAlt: 画像の説明
heroImageCredit: 画像:○○株式会社プレスリリースより
```

## 文体・構成

- です・ます調。地域住民に語りかける親しみやすいトーンで、煽りや誇張はしない。
- 本文400〜800字程度。導入（何が起きたか）→ 見出し付きの詳細 → 地域への影響や周辺情報、の構成。
- 見出しは `##` を使う。1記事に1〜3個程度。
- タイトルには「登戸」「向ヶ丘遊園」など対象の地名を含める（ローカルSEO対策）。
- 日付は「2026年3月25日（水）」のように和暦なし・曜日ありが望ましい。

## やってはいけないこと

- 画像ポリシーに反する画像利用（上記参照）
- 店舗・個人への批判、未確認の閉店・不祥事情報の掲載
- 口コミサイトの評価点数の転載（「人気です」程度の表現に留める）
- 既存記事と同じネタの再掲（`data/coverage-log.json` と `src/content/posts/` を必ず確認）

## 対象エリア

登戸駅・向ヶ丘遊園駅の周辺（川崎市多摩区）を中心に、宿河原・中野島など隣接エリアも可。それより遠い話題は、登戸住民への影響が明確な場合のみ（例: 小田急線・南武線のダイヤ改正）。

## 新しいニュースがない日の運用: アーカイブ記事（過去記事）を書く

新しいニュースが見つからない日は、**過去の出来事を記事化してアーカイブを充実させる**（サイトの検索資産を積み上げる戦略）。

- ネタ例: 過去数か月〜数年の開店・閉店、再開発の節目、定番店の紹介、「向ヶ丘遊園の歴史」のような読み物
- `pubDate` は**その出来事が起きた時期の日付**にする（例: 2026年3月25日開業の話題なら `pubDate: 2026-03-25`）。今日の日付を付けない
- 執筆時点の視点が混ざらないよう注意する（pubDate時点で書かれたように読める文章にする）。ただし事実の正確性が最優先で、出典必須のルールは通常記事と同じ
- 既カバーのネタは避ける（`data/coverage-log.json` を確認）
- 過去記事も1回の実行で1本まで

### イベント告知記事の日付ルール

イベントの告知記事は**イベント開催日より前の日付**を付ける（読者目線で「告知」として機能した形を残すため）。告知投稿・告知記事が出た日付が分かればその日付、不明なら開催日の約1週間前。イベント終了後に扱う場合も、開催前の告知として書くか、開催当日の日付でレポート風に書くかを内容に応じて選ぶ（未来の出来事を過去形で書かない・過去の出来事を未開催として書かない、の整合性だけは必ず守る）。
